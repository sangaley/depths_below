# Module Catalog

Every buildable module currently in `building/registry.rs`. Generated by parsing the registry directly, not hand-typed — counts and sizes below are exact as of this writing (155 total: 145 originally shipped + 10 shaped modules added this session, one of which — Corner Armor Plate — was the original proof-of-concept and the other nine are the properly function-justified set).

**Size** is the grid bounding box in cells (66 units/cell). **Function** is the `CompanionData` variant that gives the module its actual behavior (`—` means it's a passive/adjacency-effect block with no companion component).

## Size distribution

| Size | Count |
|---|---|
| 1x1 | 134 |
| 2x1 | 12 |
| 2x2 | 3 *(Corner Armor Plate, Galley Mess, Bulk Cargo Hold — all L-shaped)* |
| 3x2 | 3 *(Bridge Wing, Surgical Bay — T-shaped; Staggered Armor Plate — S-shaped)* |
| 3x3 | 3 *(Fusion Reactor, pre-existing; Docking Hub, Wellness Hub — plus-shaped)* |

The rule this session settled on, backed by the original data: **91% of the game is still 1x1, and shape/size only exists where it's earned by what the module does** — not decoration. See "Shaped modules" below for the reasoning per module.

## Shaped modules — what shipped and why

Each of these is a *new, additional* module alongside its plain-square counterpart (Helm Station, Med Bay, Mess Hall, Large Cargo, Docking Port, Training Room, Armor Plate all still exist unchanged) — new types rather than resized existing ones, so nothing already placed in a saved blueprint or AI ship layout breaks.

| Module | Shape | Category | Why this shape |
|---|---|---|---|
| Corner Armor Plate | L | Structural | Wraps a hull corner, leaves the notch cell free for a neighbor. |
| Bridge Wing | T | Control | Wings extend past the hull for lateral visibility — real warship bridge-wing design, built for exactly this. |
| Surgical Bay | T | Crew | Narrow triage entry into a wide treatment bay, instead of one cot. |
| Galley Mess | L | Crew | Long galley run + dining nook in the bend — where the word "galley" comes from in the first place. |
| Bulk Cargo Hold | L | Storage | Fills a leftover hull corner instead of wasting it, like real cargo-ship hold design. |
| Docking Hub | Plus | Utility | Multiple simultaneous docking connections from different sides — same idea as an ISS connecting node. |
| Wellness Hub | Plus | Crew | Same hub logic, gym/training flavor — mirrors the ISS's own exercise-module-as-node design. |
| Staggered Armor Plate | S/Z | Structural | Offset seams — no single straight line runs through the plate. |
| Angled Hull Plate | Wedge (1x1, visual only) | Structural | Tapered framing so a hull doesn't have to read as stacked boxes. |
| Angled Armor Plate | Wedge (1x1, visual only) | Utility | Tapered armor face. No deflection mechanic yet — see below. |

## Still open / deferred

- **Sloped-deflection ricochet mechanic.** The two angled/wedge pieces are visual only right now (a triangular silhouette on an otherwise normal 1x1 hitbox) — no hit-angle-vs-surface-normal calculation exists yet. Full player-customizable angle selection ("pick whatever angle you want for ricochet") is planned as a later, separate system — it needs real changes to the damage/projectile code, not just the footprint mechanism built this session.
- **Staggered Armor Plate's seam bonus is narrative only right now** — it has HP scaled up for its 4-cell size, but there's no code checking whether two staggered plates are actually adjacent/offset to grant an extra bonus. Worth deciding whether that's worth wiring up or whether the HP/cost economics alone are enough.
- **Conveyors/pipes remain deferred**, as decided earlier — `ConveyorTube` stays a plain 1x1 for now.
- **Weapon-chain shapes** (T/L ammo feeds, barrel clusters) still blocked on the `connections.rs` single-cell-registration fix, as originally scoped.
